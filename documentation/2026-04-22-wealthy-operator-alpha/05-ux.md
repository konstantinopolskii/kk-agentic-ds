## Reorganizations vs conservative default

Eight moves. Each one names the conservative default, the UX delta, and the ergonomic job it serves better.

### R1 — Inspector stack reorder: Stages → Threads → Future-reserved

**Conservative default:** Stages card top, Future-reserved middle, Threads bottom (concept's literal order).

**UX delta:** At review state (and draft onward), inspector stack renders **Stages (minimized) → Threads (primary) → Future-reserved (footer)**. Stages card adopts `card--interactive[data-state="minimized"]` once operator passes step 3 — it collapses to heading + a one-line summary ("Step 5 of 7 · Review") and keeps the full seven-row `doc__spec` behind `card__collapsible`. Promote to `data-state="active"` on click.

**Ergonomic job:** During review the operator works the threads. Stages is reference, not focus. The kit `card--interactive` minimized pattern exists for exactly this. Putting reference above primary work forces the eye to jump past noise every time a thread update comes in. Flipping the order + collapsing stages lets threads sit in the attention zone (top of inspector) where selection-originated drafts already pin.

### R2 — Control block becomes `card--interactive` with scope ladder

**Conservative default:** Static card, one button "Redo draft". Flat. Matches the concept drawing literally.

**UX delta:** Per-section control block is `card--interactive[data-state="minimized"]`. Heading row carries `N threads · Improve in place` (primary button, gate amendment's DEFAULT). On click → `data-state="active"` reveals `card__collapsible` carrying two secondary buttons: `Redo section` + `Redo whole doc` (gate amendment's rare + rarest). Research control block mirrors the pattern: primary `Implement comments in research`, collapsible reveals `Redo research` + `+ Add research`.

**Ergonomic job:** Gate amendment 2 names three scopes with explicit frequency — default / rare / rarest. Flat three-button equality contradicts the frequency spec. Kit's `card--interactive` gives free visual weight to the default (primary button, always visible) and demotes the rare actions behind a promote (collapsible reveal) — exactly the frequency hierarchy the amendment mandates, zero new components. One primary per card rule still holds: primary is always the in-place variant.

### R3 — Brief card reshapes primary scent around transcription

**Conservative default:** Four equal-weight fields — transcription, CV, mentor notes, depth switch — in row order.

**UX delta:** Transcription renders as a `field field--row` with `t-caption--bold` label (goal auto-derives from it — primary scent). CV, mentor notes, depth switch render with plain `t-caption` labels. Visual hierarchy inside the card is: transcription first row, hairline, secondary inputs below in one group. No new class — weight difference comes from existing `t-caption--bold` vs `t-caption`.

**Ergonomic job:** Concept spec: "Goal auto-derived from transcription (no goal field)." Transcription is the load-bearing input; the others refine. Equal weight hides that. The kit already has a bold-vs-regular distinction step (manifesto: one step per pair). Using it here points the operator at the field that matters.

### R4 — Sidebar grows thread-density glyph per nav item

**Conservative default:** Sidebar nav = section labels only, scroll-spy indicator.

**UX delta:** Each `nav-group__items li` with live threads appends a `t-micro t-muted` count: `Позиционирование  1`. Zero-thread rows render the label alone. The glyph is a muted micro caption — allowed because thread count is metadata (explicit manifesto carve-out for muted), not body text.

**Ergonomic job:** Operator reviewing a 9-subsection strategy needs to know where the heat is without scrolling the doc. The kit sidebar already renders a TOC; adding a metadata micro column per row turns the map into a heat map without inventing a component. Scroll-spy still owns the active-row indicator. Muted-default forbid rule is satisfied — thread count is metadata, which manifesto allows muted.

### R5 — Flat thread stack, no grouping, kit v0.13.0 kebab

**Conservative default:** Nothing to reorganize — concept already specifies flat by gate amendment 1. Both conservative and UX honor this.

**UX delta (shared with conservative):** `.comment-stack` contains `.card.card--interactive.comment-thread` in pin order. No section sort, no collapsible groupings, no cluster bins. Each `.comment__menu-popover` carries kit v0.13.0 items: Reply, Approve, Archive, Delete.

**Ergonomic job:** Flat order is operator order. Grouping by anchor section hides pin sequence and collides with the context-stream model (all open threads feed every regeneration). Call this out explicitly so stage 7 sees it and stage 8 does not bolt on grouping later.

### R6 — Past doc sections collapse via `card--interactive[data-state=minimized]`

**Conservative default:** Concept says "collapsed" past sections at review state. Conservative default renders a static card with collapsed text. Recoverable only via scroll.

**UX delta:** `## Brief`, `## Research brief`, `## Research 1` sections each wrap their content in `card--interactive[data-state="minimized"]`. Heading stays legible with a one-line summary spec row (`Sofia · deep · Nelli call · CV`). Promote button opens the full card inline (kit default). Kit manifesto: one active per stack — here the "stack" is the doc body sequence; kit rule still applies, at most one past section active at a time.

**Ergonomic job:** The accumulative-doc direction wants every past stage visibly survived, but scan-time degrades if nine past sections render in full. Kit's minimized interactive card is the exact mechanic: content preserved, scent compressed to one line, one-click recovery. No invention.

### R7 — Signoff shout stays at bottom; step list legibility via `t-caption--bold` on active step

**Conservative default:** Signoff shout at bottom. Steps render as a flat list of `t-caption` rows. Current step has a `●` glyph.

**UX delta:** Step list keeps kit inventory (`card__collapsible` with `t-caption` rows inside `card--shout`). Active step row uses `t-caption--bold` (not a new class; existing utility). Future steps stay `t-caption`. Satisfied (past) steps stay `t-caption` + `✓` glyph. The distinction is one step: bold vs regular, not bold + muted + color.

**Ergonomic job:** Manifesto: "one distinction step between two elements." Active vs rest is the signal the operator needs. Using `t-caption--bold` follows the single-step rule and stays in inventory. No promoting the step list to the top of the doc — one shout per column stands, and the step list without the Sign button lacks closure.

### R8 — Stages card content: `doc__spec--value` (three columns) to carry state + substate

**Conservative default:** Two-column `doc__spec` — stage name + ✓/●/·.

**UX delta:** `doc__spec--value` (three-column kit variant). Column 1: stage name (`t-caption`). Column 2: state glyph (`✓ ● ·`). Column 3: micro summary when helpful (`Rsrch brief ✓ · 8 sources` on research row; empty on not-yet rows). The third column is a kit-native slot — already in `components.md`.

**Ergonomic job:** Concept calls for past-stage summaries ("2 threads accepted, 1 rejected"-style). Two-column can't carry that. The three-column spec variant is kit inventory and was designed for this class of data. No new component, no extra card.

## Behaviors

Kit defaults everywhere. New compositions named explicitly.

### Shell

- `.app[data-view="doc"]` — standard three-column. `KK.init()` on DOMContentLoaded.
- `.sidebar__nav` — kit scroll-spy (`toc__indicator` animates on active section change). Per-row thread glyph is static markup the consumer updates via its own state; no kit hook needed.

### Brief input card

- `.card` static until operator commits. Fields inside: `.field.field--row` × 4. Transcription field uses `.field__label` with `t-caption--bold`; others `t-caption`.
- `.switch` for depth. Kit binary toggle. Label pair: shallow | deep.
- Commit button: `.button.button--primary` labeled at stage 9 (copywriter).
- Post-commit: card flips to `.card.card--interactive[data-state="minimized"]`. Heading row renders `Sofia · deep · Nelli call · CV` (`t-caption t-muted` summary — muted allowed, this is metadata). Minimized CTA: secondary "Edit brief"; primary "Commit brief" hides into collapsible (per kit "primary and minimized button labels never match").

### Control blocks (per-section, gate-amendment scope ladder)

- `.card.card--interactive[data-state="minimized"]` by default.
- Heading: `t-caption` carrying `N threads` (left) and `Improve in place` primary CTA (right, `button--primary`).
- `card__collapsible` holds `Redo section` + `Redo whole doc` as `.button` secondaries, stacked.
- Research variant: primary `Implement comments in research`, collapsible carries `Redo research` + `+ Add research`. `Redo research` confirms via native confirm modal — **flagged to stage 8**: kit has no modal component, and gate amendment says `Redo research` needs confirmation because the research run is replaced. Stage 8 uses browser `confirm()` or defers to stage 6 (revolutionary) to propose a modal. Conservative path: the primary action is non-destructive (`Implement comments`), and the destructive secondary is labelled strongly — behavioral risk is bounded.
- `N threads` text is a link (`<a>` inside `t-caption`) that scrolls the inspector `.comment-stack` to the first open thread anchored at the owning section. Uses standard anchor behavior, no kit JS addition.

### Past-section minimized cards (R6)

- Applies to `## Brief`, `## Research brief`, `## Research 1` once operator passes that stage.
- `.card.card--interactive[data-state="minimized"]`. Heading = section title + one-line summary. CTA to expand uses secondary button. Only one past section active at a time (kit rule).

### Stages card (inspector top)

- Steps 1-3: `.card.card--interactive[data-state="active"]`. Full seven-row `doc__spec--value` visible.
- Step 4 onward: collapses to `.card.card--interactive[data-state="minimized"]`. Heading carries current stage as text (e.g., `Step 5 · Review`). Promote on tap.
- Rows inside: `.doc__spec.doc__spec--value`. Key = stage label, middle = glyph, right = metadata micro.

### Thread stack (inspector primary zone)

- `.comment-stack` flat.
- Each thread: `.card.card--interactive.comment-thread`. One active at a time.
- `.comment__menu-popover` per `.comment-msg` with kit v0.13.0 items: Reply, Approve, Archive, Delete.
- Draft flow: operator selects doc text → kit's `KK.enableCommentSelectionFlow()` pins `.comment-new` as `.card--shout` at top of stack. Shout rule holds: one shout per column — inspector has no other shout (signoff is in doc column).
- Context stream (gate amendment 4): every regeneration reads all open threads. Kit event `kk:comment` already fires on new/reply/delete; kit v0.13.0 adds approve/archive. Consumer binds once.
- Lifecycle (gate amendment 5): agent reply lands as a `.comment-msg` inside the thread, styled as the kit default. Operator kebab → Approve (consumer swaps message into doc body, thread enters resolved state), Reject-with-answer (operator types reply, thread stays open next round), Leave unanswered (skip; still fed to context). Archive via kebab hides thread. Resolved + archived both stop traveling (amendment 6).

### Signoff shout (doc bottom)

- `.card.card--shout` with `.card__heading`, `.doc__signoff-stats` horizontal row (revs / hits / edits pending), and a step list. Step list uses `.card__collapsible` (always open in this card — no collapse affordance; it reads as a body list).
- Active step row: `t-caption--bold`.
- Satisfied step row: `t-caption` + `✓`.
- Pending step row: `t-caption` + `.` glyph.
- Primary button: `Sign` (`.button.button--primary`). Enabled whenever strategy draft exists (guidance not gate, per concept rabbit-hole note).
- On sign → kit class swap drives the resort animation. No keyframe invention beyond kit motion tokens.

### Delivered state shift (post-sign)

- Doc body resorts. Signoff shout evaporates; its content merges into a static title block `.card` at doc top with `doc__signoff-stats` + `doc__signoff-signature`.
- Inspector stages card stays (now minimized at Delivered row). Threads stack empties (all resolved). Future-reserved slot swaps to Delivered card (share URL + Copy + Patch) + Edit log card.
- Nav relabels in one hard swap on resort endpoint.

## Reasoning

One paragraph per major choice. Principle named.

**Inspector stack reorder (R1).** Principle: 80/20 + pure signal. At review state, threads are the job and stages are reference. Stacking reference above primary work taxes every glance. Kit already gives a minimized interactive card for exactly the "reference that should compress once consumed" case. Pushing stages to minimized once operator passes step 3, and raising threads to the attention zone, costs no invention and reclaims the top inspector real estate for the actual job.

**Control block as scope ladder (R2).** Principle: expected patterns + chunking. Gate amendment 2 mandates three scopes at different frequencies — default, rare, rarest. Three equal buttons contradict the frequency. The kit's `card--interactive` pattern is the native mechanism for primary-visible / secondary-hidden — the primary surfaces the default, the collapsible reveals the rare cases. One primary per card still holds. No invention. The scope ladder becomes legible at a glance.

**Brief primary scent on transcription (R3).** Principle: one distinction step. Concept spec: goal derives from transcription. The other three fields refine. Making all four labels the same weight hides the load-bearing input. `t-caption--bold` on the transcription label, `t-caption` on the others, is the kit's one-step-per-pair rule applied literally. No color, no size change — weight alone.

**Sidebar thread-density glyph (R4).** Principle: pure signal + muted-for-metadata exception. Operator scanning a 9-subsection strategy needs heat location without scrolling. The sidebar is already a map; adding a metadata column to each nav row turns it into a heat map. Muted is allowed for metadata (manifesto carve-out). The kit sidebar nav already uses `t-caption` labels — appending a `t-micro t-muted` count sibling is composition, not invention.

**Flat threads (R5).** Principle: expected patterns + gate amendment 1. The kit's Comment component renders threads in pin order inside `.comment-stack`. Gate amendment explicitly forbids grouping. Call it out so stage 8 does not invent grouping when thread count rises.

**Past sections via minimized interactive card (R6).** Principle: signal / noise + the iPad feel. The accumulative-doc direction wants past stages survived; scan degrades at nine full sections. Kit's `card--interactive[data-state="minimized"]` is the mechanic — content preserved, scent compressed to a heading row, one click to recover. Manifesto: only the middle column scrolls during reading. Keep that, and give the reader the right density at each stage.

**Signoff shout stays at bottom (R7).** Principle: one shout per column + expected patterns. The Sign button has to live where the steps live or the closure reads broken. Promoting the step list to the top as a second card splits the affordance and invites a second shout (forbidden). Bold on the active step is the one-step distinction. Signoff lives where the operator ends the doc.

**Stages card as three-column spec (R8).** Principle: chunking + 80/20. Past stages carry context (source count, thread count summary). Two-column spec can't fit that without truncation. Three-column is kit inventory (`doc__spec--value`) — already designed for key/value/explanation triples. Using it means the stages card carries its information load without a new component.
